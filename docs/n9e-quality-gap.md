# n9e 1:1 品质差距分析

> SREAgent 已实现但做得不如 n9e 好的地方，按模块逐一对比
> 原则：不局限前后端，只关注「同等功能」下的品质差异

---

## 1. 告警规则管理

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **规则列表** | 分类 Tab + 名称搜索 + 数据源筛选 + 启用/禁用快速切换 | 仅名称搜索 + 分页 | 缺分类过滤和筛选组合 |
| **规则表单** | 数据源分组选择器、标签建议（基于已存在 label key/value）、表达式测试按钮 | 纯文本输入、无自动补全 | 缺少 label 建议和表达式测试入口 |
| **规则模板** | 预定义模板库，用户选模板改阈值即可 | 从零创建每条规则 | 缺模板系统 |
| **批量操作** | 批量启用/禁用/删除、批量修改标签/注解 | 无批量操作 | 缺批量操作 |
| **版本对比** | 前后版本 diff 对比视图 | 仅列出历史版本，无 diff | 缺版本 diff |
| **规则分组** | 支持按业务组/data center 多维度分组展示 | 平铺列表 | 缺分组视图 |
| **导入导出** | 支持 YAML/JSON 格式、支持跨数据源导入 | 仅 Prometheus YAML 导入导出 | 格式单一 |

## 2. 告警事件

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **事件列表** | 多条件组合筛选（严重程度/状态/数据源/时间/标签）、列自定义显示/隐藏 | 基础筛选（状态+严重程度）、固定列 | 筛选能力弱 |
| **批量操作** | 批量认领/分派/静默/关闭、支持选中的和筛选结果集两种模式 | 仅全选+批量认领/关闭 | 缺批量分派/静默 |
| **事件详情** | 生命周期时间线 + 通知记录 + 告警时原始数据 + 相关告警推荐 | 时间线 + 评论 | 缺通知记录追踪、缺原始数据查看、缺关联告警 |
| **事件列表自动刷新** | WebSocket 实时推送 | 30s 轮询 | 实时性差 |
| **事件导出** | 支持 CSV/JSON、自定义导出列 | 仅 CSV、固定列 | 导出灵活性差 |
| **标签展示** | 标签以 badge 形式展示、支持点击筛选/复制 | 仅字符串拼接 | 交互弱 |
| **已恢复/已关闭分离** | 历史告警独立页面，支持按时间范围查询 | 混在事件列表用状态切换 | 分离度不够 |

## 3. 仪表盘

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **面板类型** | 时序图 / Stat / Gauge / Bar / Pie / Table / Heatmap / Text 共 8+ 种 | 仅时序图 + Table | 缺 6 种面板 |
| **面板布局** | 拖拽 Grid 布局，自由调整位置和大小 | 固定纵向排列 | 无可视化布局编辑器 |
| **变量系统** | query/custom/textbox/constant/interval 共 5 种 | query/custom/textbox/constant 共 4 种 | 缺 interval 类型 |
| **变量联动** | 变量间可设置级联关系（如选 DC → 选集群 → 选主机） | 独立变量、无级联 | 无级联能力 |
| **时间范围** | 支持 per-panel 独立时间范围 | 全局统一时间范围 | 缺面板级时间覆盖 |
| **图表交互** | 图表点击 → 下钻到详情、图例点击 → 隐藏/显示系列、tooltip 显示完整标签 | 基础 tooltip + legend 切换 | 交互不够丰富 |
| **仪表盘管理** | 收藏、标签分类、按分组共享、导入导出 JSON | 仅 public/private | 组织能力弱 |
| **门槛** | 可直接在面板上修改 PromQL | 需要进入编辑模式 | 操作路径长 |

## 4. 数据源管理

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **数据源类型** | Prom / VM / ES / Loki / CK / InfluxDB / Zabbix / 自定义 HTTP 共 8+ | Prom / VM / VLogs / Zabbix 共 4 | 缺 ES/Loki/CK/InfluxDB |
| **连接测试** | 创建/编辑时一键测试、实时状态展示（healthy/unhealthy/reachable 等） | 有健康检查但 UI 入口不直接 | 测试不够便捷 |
| **类型专属表单** | 不同类型有不同的配置字段（如 ES 需要 index pattern、CK 需要 cluster 名） | 统一表单 + 手动填 AuthConfig | 配置体验差 |
| **标签自动发现** | 自动扫描 label keys/values 并提供自动补全 | 有 label registry 但自动补全仅在 Explore 页 | 补全未在规则表单中复用 |
| **查询页面** | Explore 页面支持多查询 Tab + 结果对比 | 多查询同面板、无对比模式 | 缺对比能力 |

## 5. 通知管理

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **通知渠道** | 飞书/钉钉/企微/邮件/短信/Telegram/Slack/Webhook/自定义脚本 共 9+ | 飞书/邮件/Webhook/Script 共 4 | 缺钉钉/企微/Telegram/Slack/短信 |
| **通知模板** | 丰富的 Go template 变量（含告警上下文、值班人信息、数据源链接等）、模板预览 | 基础 template 变量 + 预览 | 变量丰富度不足 |
| **通知记录** | 每次通知发送结果可追溯（成功/失败/重试/耗时） | 无单独的通知发送记录 | 通知可观测性差 |
| **用户订阅** | 用户可按告警规则/标签/严重程度自主订阅 | 仅按标签匹配 | 订阅粒度粗 |
| **通知策略** | 多级通知 + 通知升级 + 通知聚合 | 有升级策略 + 分组通知 | 持平（SREAgent 的 Pipeline 更灵活）|

## 6. 值班排班

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **排班视图** | 日历视图 + 甘特图 + 列表视图 | 仅列表 + 基础日历 | 缺甘特图/时间轴视图 |
| **排班生成** | 一键生成下一周期排班、可视化调整后保存 | 手动创建每个 shift | 缺少批量生成 UI |
| **替班管理** | 可视化交换两个值班人的时间段 | API 存在但 UI 未完全实现 | UI 缺交换/替班可视化 |
| **值班提醒** | 值班前 N 小时自动提醒 | 无提醒 | 缺值班前提醒 |
| **iCal 导出** | 一键复制链接/下载 | 有但入口不够明显 | 入口隐蔽 |

## 7. 静默 / 抑制规则

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **静默规则时间** | 可视化时间范围选择器 + 周期性 Cron 配置 | 输入框手动填周期 | 配置体验差 |
| **静默预览** | 实时展示当前匹配的告警列表 | API 有但前端无预览 | 缺匹配预览 UI |
| **抑制规则调试** | 模拟输入标签 → 显示是否被抑制及原因 | 无调试工具 | 缺调试能力 |
| **规则顺序** | 可拖拽调整抑制规则优先级 | 固定按创建顺序 | 缺优先级调整 |

## 8. RBAC / 权限

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **Busi Group 集成** | Busi Group 是核心权限边界：用户属于 Group，在 Group 内分配角色 | Biz Group 更像标签作用域，与用户没有强绑定 | 权限模型未 Group 中心化 |
| **数据源可见性** | 数据源按 Busi Group 隔离 | 全部用户可见所有数据源 | 缺数据源级别隔离 |
| **仪表盘可见性** | 仪表盘按 Busi Group 共享 | 仅 public/private | 缺 Group 级共享 |
| **角色自定义** | 支持自定义角色 + 权限点组合 | 固定 5 种角色 | 角色不可自定义 |

## 9. 前端 UI/UX

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **导航结构** | 两级侧栏：一级图标视图 + 二级展开面板（类 VS Code 活动栏） | 传统单级侧栏 | 导航信息密度低 |
| **列表页交互** | 行 hover 露出操作按钮、行点击展开详情/侧边抽屉、Shift/Ctrl 多选 | 操作按钮始终可见、点击跳转详情页 | 交互效率稍低 |
| **表单体验** | 分组折叠面板、字段级校验实时提示、自动保存草稿 | 模态框表单、提交时校验 | 表单体验一般 |
| **空状态** | 每个页面有空状态插图 + 引导操作按钮 | 基础 NEmpty 组件 | 缺引导性空状态 |
| **加载状态** | 骨架屏 (Skeleton) 替代空白 | 大部分用 NSpin | 缺骨架屏 |
| **暗色模式** | 完整暗色主题，包括图表配色自适应 | 有暗色但 ECharts 和部分组件硬编码颜色 | 暗色覆盖不完整 |
| **移动端** | 响应式布局，移动端可用 | 未做移动端适配 | 移动端不可用 |
| **快捷键** | 全局 Ctrl+K 搜索 + 页面级快捷键 | 有 Ctrl+K 但功能简单 | 快捷键体系弱 |
| **错误处理** | 表单错误 inline 提示 + API 错误自动重试 | 仅右上角 message 提示 | 错误提示不够就近 |

## 10. 后端健壮性

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **引擎状态持久化** | Redis + MySQL 双重保障，Redis 不可用时自动降级 | 仅 Redis（代码审查已发现） | Redis 故障容忍度低 |
| **Webhook 认证** | 共享密钥验证 + 来源 IP 白名单 | **无任何认证**（Critical Bug） | 安全缺口 |
| **Goroutine 管理** | Worker Pool 限制并发数 | 无限制创建 goroutine（每次告警一个） | 大量告警时可能 OOM |
| **自监控 /metrics** | 内置 Prometheus metrics 端点 | **无** | 缺乏可观测性 |
| **API 限流** | 内置令牌桶限流器 | 无 API 限流 | 缺少保护 |
| **优雅关闭** | 等待所有在途请求完毕后关闭 | 关闭顺序有问题（Evaluator 在 HTTP Server 之后停） | 关闭时可能丢告警 |
| **审计日志** | 自动记录所有 CRUD + 登录 + 导出操作 | 有但覆盖不全面（如导出操作无审计） | 审计粒度粗 |

## 11. 测试

| 维度 | n9e | SREAgent | 差距 |
|------|-----|----------|------|
| **单元测试** | 各模块有基础测试覆盖 | 仅 2 个骨架测试文件 | 22/24 模块 0% 覆盖率 |
| **集成测试** | 有端到端 API 测试 | 无 | 缺集成测试 |
| **前端测试** | 无 | 无 | 持平 |

## 优先级修复路线图（按「做了就明显比 n9e 好」排序）

### P0 — 安全/稳定性（必须修）
1. Webhook 认证 → 加 shared secret 验证
2. Goroutine Pool → Worker Pool 限制并发
3. 优雅关闭顺序 → 修 main.go shutdown 流程

### P1 — 体验质的飞跃（1 周内可完成）
4. 告警规则批量操作（批量启用/禁用/删除）
5. 事件列表 WebSocket 实时推送
6. Stat 面板 + Gauge 面板（最常用的 2 种面板）
7. 通知发送记录追踪
8. 健康检查一键测试按钮

### P2 — 补齐到 n9e 水平（2-3 周）
9. 仪表盘面板拖拽布局
10. 告警规则模板库
11. 钉钉/企业微信通知渠道
12. Busi Group 用户绑定 + 数据源/仪表盘可见性隔离
13. 暗色模式 ECharts 适配
14. Elasticsearch 数据源

### P3 — 超越 n9e（长期）
15. AI 根因分析（n9e 没有）
16. Pipeline 高级处理器（n9e 没有）
17. 移动端 PWA
18. 多租户隔离
